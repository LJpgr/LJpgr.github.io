## 什么是数据结构和算法？

数据结构就是一组数据的存储结构，算法就是操作数据的一组方法。

例如，图书馆中的书籍如果比作数据的话，那么各个书架就是存储数据的“数据结构”，如何找到一本书所需要的方法就称为算法。

在计算机的角度观察，数据结构就是存储数据的逻辑结构，而非物理结构，例如，栈、序列、队列、树等等，相应地，算法就是在这些存储结构中查询、修改、删除、添加对应数据的方法，如：冒泡排序、二分查找、动态规划等。

数据结构与算法之间是不可分割的，算法就是作用于对应的数据结构之上，数据结构就是为算法提供的基础逻辑，可以说两者是相辅相衬，互相影响的。

## 如何衡量一个算法的好坏?

既然存在这么多的算法，那必然其中有好有坏。简单来说，算法的优劣与程序执行的时间成反比，时间越短，算法则优，反之亦然。不过，这个规律不同的外在条件可能会有不同的结果，比如，数据规模，硬件条件，操作系统等等。

那么，如何通过具体的指标来衡量一个算法呢？最简单的比较算法执行效率的方法就是直接判断程序执行的次数。假设排除其他外在因素的影响，程序每一行执行的时间都为==utime==。参考下边代码：

```java
int A(){
	int a=0;
	int b=10;
	for(int t=0;t<b;t++){
		a+=t;
	}
	return a;
}
```

第2，3，7行代码只需要执行1次，第4，5行代码需要执行b次，也就是执行10次，那么这段代码最终的执行时间T=(3+2\*b)\*utime,也就是 需要 耗时23utime 。

我们由此可以归纳总结出规律，程序执行的时间与代码执行次数成正比。可以得出一下公式：
>
T(n)=O(f(n))

其中T代表程序执行的时间，n代表数据的规模，f(n)代表程序执行的次数总和， O 为时间与执行次数的关系，是一种随着数据规模变化相对应时间变化的趋势，这种表示法就是**<u>大 O 时间复杂度表示法</u>**。

例如下边代码：

```java
int A(int n){
    int a=0;
    int b=0;
    for (int t=0;t<n;t++){
        a+=1;
    }
    for (int i=0;i<n;i++){
        for (int j=0;j<n;j++){
            b+=1;
        }
    }
}
```

此代码中第2，3行只执行1次，对于时间复杂度 O 为 O(2)=C=1，我们认为是常数时间内执行，几乎不影响复杂度 O。第4，5，7行执行了n次，时间复杂度为 O(n) 。第8，9行代码执行了n*n次，时间复杂度为 O(n<sup>2</sup>)。那么该段程序最终的时间复杂度为 O(n<sup>2</sup>)+O(n)+C，从多项式的函数图像来看，影响函数曲线斜度最大的是最高次项，其他项可忽略不计，最终该段代码的时间复杂度为 O(n<sup>2</sup>)。

- 只需要关注循环次数最多的一段代码
- 整体复杂度总是等于量级最大的那一段代码
- 嵌套代码的复杂度等于两者复杂度的乘积

## 还有什么其他方法？

除了从时间的角度分析算法的效率，还可以从空间的角度分析。类似地，空间复杂度即随着数据规模的变化存储空间的变化趋势。

看下段代码示例：

```java
void execute(int n){
    int a=6;
    int arr=new int[n];
    for (int i=0;i<arr.length;i++){
        arr[i]=a;
    }
}
```

这段代码第2行只需要申请一个变量的空间，记为 O(1)。第3行代码申请了存放n个int类型的数组，使用的空间复杂度记为 O(n)。其他行代码并没有使用额外的空间。所以，这段代码的空间复杂度为 O(n) 【忽略常数项】。
